 Проект создан для возможности вклиниться в поток передаваемых данных, посредством COM портов.
 Идеей было запуск этой программы на Raspberry pi или подобный, к которой к двум COM портам
 подключены общающиеся между собой устройства. Программа может пересылать все входящие сообщения
 с первого порта на второй и со второго на первый, сохроняя лог общения. На практике выяснилось, что из-за
 большого пинга с одной из сторон мообщение может биться по частям произволным образом, приходящими с
 небольшими задержками. В связи с этим реализованы два режима ретрансляции:
 1. "по байтам" - все принимаемые байты тут же ретранслируются из указанного порта.
 2. Просто ретрансяция или ретрансяция сообщений - все принимаемый байты сохраняются в буффер, пока
 разница между их приходами достаточно мала. При долгом отсутствии данных, принятый буффер нарекается сообщением и,
 при необходимости пересылается из указанного порта.
 В результате вышла платформа с возможностью добавить неограниченное количесвтво COM портов и организовать
 происзовльную ретрансляцию между ними.

 Здесь и далее (в коде и комментариях) если явно указывается пересылка "по байтам" - имеется ввиду она, в противном
 случае подразумевается пересылка сообщений, сформированных в буфере.

 Так же надо понимать, что ретрансляция происходит не в порт, а из порта. Программу можно воспринимать как коробку,
 с множеством COM портов, и то что она получает в порт А - может выкидывать из порта В.

 Таким обазом, если вы, подключите один провод в два COM порта (или с исользованием переходников USB - COM,
 распознаваемых системой как COM), назначите ретрансляцию из порта 1 в порт 2 и отправите что-то из второго порта -
 оно приде на первый и начент бесконечно бегать по кругу.


 По команде help, программа выводит справку с подробным описание воспрнимаемых комманд:

                                Welcome to Sniffer2!
             List of commands:
     >list                                 -   list of com-ports
     >add <com mane>                       -   add to sniffer com port with name <com name>
     >listen <com name>                    -   start to listen com port <com name>
     >send <com name> <massage>            -   send from <com name> port massage <massage>
     >setRet <com name 1> <com name 2>     -   all accepted massage from <com name 1> port
                                               sending in <com name 2>. Work with buffer by 5 ms
     >setRetBB <com name 1> <com name 2>   -   all accepted bytes from <com name 1> port sending
                                               in <com name 2>. Resending by byte, without buffer
     >setNoRet <com name>                  -   set off all retransmitions
     >close <co name>                      -   close port and remove from Sniffer2
     >setOut <file name>                   -   set out of listeners to file <file nam>
     >help                                 -   print help, about Sniffer2
     >exit                                 -   close all ports and exit
         Good luck, your Sniffer!


 Программа исользует нативную библиотеку jSSC.
